# Replication: Imbalanced Learning Efficiency and Cognitive Effort in SUD

Computational replication of the reinforcement learning models from:

> Fang, Gao, Xia, Cheng et al. (2026). *Imbalanced learning efficiency and cognitive effort in individuals with substance use disorder.*

The paper fits 8 RL models to multi-task learning behavior in substance use disorder (SUD) vs. healthy controls (HC), finding that the Resource-Rational Successor Features model (RRSF) best accounts for the data.

---

## Repository Structure

### [`env.py`](env.py) — Task environment

Defines the shared task world as a static-method class `Env`. The environment is a 10-state deterministic tree:

- **States**: s=0 (root) → s=1–3 (intermediate) → s=4–9 (terminal)
- **Features**: `Env.PHI[s]` — 3D resource vector φ(s); zero for non-terminal states
- **Goals**: `Env.GOALS` — price vectors w_g for 4 training tasks + 1 test task
- **Transitions**: `Env.TRANSITIONS[s][a]` — deterministic next state
- **Reward**: R = w_g · φ(s), non-zero only at terminal states
- `Env.make_trial_sequence()` generates the 80-trial training sequence (4 goals × 20 trials each)

### [`models/`](models/) — RL model implementations

All models expose a uniform interface:

```python
model.simulate(trial_sequence, params, pi0_init, rng) -> list[int]
model.log_likelihood(actions, trial_sequence, params, pi0_init) -> float
```

**Framework 1 — Classical RL** (maximize expected reward):

| File | Model | Description |
|------|-------|-------------|
| [`mb.py`](models/mb.py) | Model-Based | Learns transition T̂ and feature d̂; recomputes Q each trial via value iteration |
| [`mf.py`](models/mf.py) | Model-Free | Goal-conditioned Q-table updated by TD error |
| [`mfp.py`](models/mfp.py) | MF + Perseveration | MF with task-agnostic action memory M(a) |
| [`sf.py`](models/sf.py) | Successor Features | Learns Ψ(s,a) once; Q_g = Ψ · w_g |
| [`sfp.py`](models/sfp.py) | SF + Perseveration | SF with task-agnostic action memory |

**Framework 2 — Resource-Rational RL** (maximize reward minus KL cost from a default policy):

| File | Model | Description |
|------|-------|-------------|
| [`rrmf.py`](models/rrmf.py) | RRMF | MF Q-values with resource-rational policy |
| [`rrsf.py`](models/rrsf.py) | **RRSF** (best model) | SF values + resource-rational policy + learnable π_0 |
| [`rrmb.py`](models/rrmb.py) | RRMB | MB planning with soft value iteration (excluded from model recovery; slow) |

**[`models/base.py`](models/base.py)** — shared utilities:
- `softmax(q, tau)` — standard softmax policy
- `rr_policy(q, pi0, beta)` — resource-rational policy: π* ∝ π_0 · exp(Q/β)
- `pack(params, spec)` / `unpack(x, spec)` — parameter transforms for unconstrained BFGS optimization
- `DEFAULT_PI0` — uniform reference policy (not used for fitting)

### [`fitting.py`](fitting.py) — MAP estimation and model comparison

Implements the fitting procedure used throughout the notebooks:

- `fit_map(model, actions, trial_sequence, pi0_init)` — MAP estimation via BFGS with 40 random restarts; returns best parameters and log-likelihood
- `compute_pi0_init(first_actions_by_state)` — computes the group-level π_0 prior from participants' first actions (must be called once before fitting any individual)
- `bic(log_lik, n_params, n_obs)` / `aic(log_lik, n_params)` — model comparison metrics

### [`notebooks/qualitative_replication.ipynb`](notebooks/qualitative_replication.ipynb) — Replication notebook

A qualitative replication via forward simulation — no MAP fitting. Virtual HC (n=45) and SUD (n=83) cohorts are generated by sampling parameters from distributions calibrated to the paper's reported group differences (SI Fig. S4), then simulating the RRSF model. The notebook is organized into six phases:

1. **Environment verification** (Phase 1) — confirms the reward matrix and task structure
2. **Parameter distributions** (Phase 2) — samples and previews HC/SUD parameter draws calibrated to SI Fig. S4
3. **Forward simulation** (Phase 3) — simulates all cohorts under RRSF, classical SF, MF, and MB
4. **Behavioral pattern validation** (Phase 4) — reproduces the four patterns of Figure 2: SUD reward deficit, Type B advantage, Room 7 co-selection, and SUD Room 4 fixation
5. **Classical model comparison** (Phase 5) — shows classical SF/MF/MB cannot reproduce Patterns 2 and 4, matching SI Figs. S2 & S3
6. **Cognitive effort analysis** (Phase 6) — computes per-trial effort = log(π/π₀), reproducing Figure 5: SUD exerts less effort, effort predicts reward, and λ predicts effort

# 复现：物质使用障碍中的不平衡学习效率与认知努力

对以下论文中强化学习模型的计算复现：

> Fang, Gao, Xia, Cheng 等 (2026). *物质使用障碍个体的不平衡学习效率与认知努力。*

该论文将 8 个强化学习模型拟合至物质使用障碍（SUD）患者与健康对照（HC）的多任务学习行为数据，发现资源理性继承特征模型（RRSF）对数据的解释最优。

```bash
uv run --with jupyter jupyter notebook      # 启动 notebook
```

## 项目结构

### [`env.py`](env.py) — 任务环境

以静态方法类 `Env` 定义共享任务世界。环境为一个包含 10 个状态的确定性树：

- **状态**：s=0（根节点）→ s=1–3（中间节点）→ s=4–9（终端节点）
- **特征**：`Env.PHI[s]` — 3 维资源向量 φ(s)；非终端状态为零向量
- **目标**：`Env.GOALS` — 4 个训练任务 + 1 个测试任务的价格向量 w_g
- **状态转移**：`Env.TRANSITIONS[s][a]` — 确定性下一状态
- **奖励**：R = w_g · φ(s)，仅在终端状态非零
- `Env.make_trial_sequence()` 生成 80 次试验的训练序列（4 个目标 × 每个 20 次）

### [`models/`](models/) — 强化学习模型实现

所有模型暴露统一接口：

```python
model.simulate(trial_sequence, params, pi0_init, rng) -> list[int]
model.log_likelihood(actions, trial_sequence, params, pi0_init) -> float
```

**框架一 — 经典强化学习**（最大化期望奖励）：

| 文件 | 模型 | 说明 |
|------|------|------|
| [`mb.py`](models/mb.py) | 基于模型（MB） | 学习转移矩阵 T̂ 和特征 d̂；每次试验通过值迭代重新计算 Q |
| [`mf.py`](models/mf.py) | 无模型（MF） | 目标条件 Q 表通过 TD 误差更新 |
| [`mfp.py`](models/mfp.py) | MF + 持续效应 | MF 加任务无关的动作记忆 M(a) |
| [`sf.py`](models/sf.py) | 继承特征（SF） | 一次性学习 Ψ(s,a)；Q_g = Ψ · w_g |
| [`sfp.py`](models/sfp.py) | SF + 持续效应 | SF 加任务无关的动作记忆 |

**框架二 — 资源理性强化学习**（最大化奖励减去与默认策略的 KL 散度代价）：

| 文件 | 模型 | 说明 |
|------|------|------|
| [`rrmf.py`](models/rrmf.py) | RRMF | MF Q 值 + 资源理性策略 |
| [`rrsf.py`](models/rrsf.py) | **RRSF**（最优模型） | SF 值 + 资源理性策略 + 可学习的 π_0 |
| [`rrmb.py`](models/rrmb.py) | RRMB | MB 规划 + 软值迭代（因速度慢，排除于模型恢复之外） |

**[`models/base.py`](models/base.py)** — 共享工具函数：
- `softmax(q, tau)` — 标准 softmax 策略
- `rr_policy(q, pi0, beta)` — 资源理性策略：π* ∝ π_0 · exp(Q/β)
- `pack(params, spec)` / `unpack(x, spec)` — 无约束 BFGS 优化的参数变换
- `DEFAULT_PI0` — 均匀参考策略（不用于拟合）

### [`fitting.py`](fitting.py) — MAP 估计与模型比较

实现整个 notebook 中使用的拟合流程：

- `fit_map(model, actions, trial_sequence, pi0_init)` — 通过 BFGS 进行 MAP 估计（40 次随机重启）；返回最优参数和对数似然
- `compute_pi0_init(first_actions_by_state)` — 从被试首次动作中计算群体水平 π_0 先验（须在拟合任何个体前调用一次）
- `bic(log_lik, n_params, n_obs)` / `aic(log_lik, n_params)` — 模型比较指标

### [`notebooks/qualitative_replication.ipynb`](notebooks/qualitative_replication.ipynb) — 复现 notebook

通过前向仿真进行定性复现，不含 MAP 拟合。通过对校准至论文报告组间差异（SI Fig. S4）的参数分布进行采样，生成虚拟 HC（n=45）和 SUD（n=83）被试群体，并使用 RRSF 模型进行仿真。Notebook 共分六个阶段：

1. **环境验证**（第一阶段）— 确认奖励矩阵和任务结构
2. **参数分布**（第二阶段）— 采样并预览根据 SI Fig. S4 校准的 HC/SUD 参数
3. **前向仿真**（第三阶段）— 在 RRSF、经典 SF、MF 和 MB 模型下仿真所有被试群体
4. **行为模式验证**（第四阶段）— 复现 Figure 2 中的四个行为模式：SUD 奖励不足、B 型任务优势、7 号房间共选择现象，以及 SUD 对 4 号房间的固着
5. **经典模型对比**（第五阶段）— 展示经典 SF/MF/MB 无法复现模式 2 和模式 4，与 SI Figs. S2 & S3 一致
6. **认知努力分析**（第六阶段）— 计算每次试验的努力值 = log(π/π₀)，复现 Figure 5：SUD 付出更少努力，努力预测奖励，λ 预测努力
